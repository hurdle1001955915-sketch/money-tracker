# 修正・改善計画まとめ

このドキュメントは、アプリの現状の課題分析（監査）に基づき、今後どのような順序で修正・改善を行っていくかを非エンジニアの方にも分かりやすくまとめたものです。

---

## 🔍 現状の診断結果

アプリ全体を詳細に調査した結果、機能は豊富ですが、**「家の土台（基本部分）」にグラつき**が見つかりました。
具体的には、以下のリスクがあります。

1. **突然アプリが開かなくなるリスク**（起動時の不具合）
2. **入力したデータが消えてしまうリスク**（保存の仕組みの弱さ）
3. **同じ機能が２つあって使いにくい**（CSVインポート機能の重複）
4. **データが増えると動作が重くなる**（計算方法の効率不足）

これらを放置して新機能を追加すると、家が傾いてしまうため、まずは**「土台の補強」**から順に進める計画を立てました。

---

## 🗓 今後の進め方（ロードマップ）

修正作業を4つのフェーズ（段階）に分けて進めます。

### ✅ フェーズ0：止血・緊急対応（最優先）

**「まずは安心して使える状態にする」**

* **何をする？**
  * アプリ起動時に失敗しても、強制終了せずにエラーメッセージを出すように修正します。
  * データ入力直後にアプリを閉じても、確実に保存されるように仕組みを強化します。
  * 外部（ログ）に個人情報（金額やカテゴリ名）が漏れないようにします。
* **効果**
  * アプリが突然落ちる恐怖がなくなります。
  * 「入力したはずなのに消えている」という事故を防げます。
  * セキュリティ面でも安心して使えます。

### ✅ フェーズ1：機能の整理（CSVインポート）

**「迷わない使い勝手にする」**

* **何をする？**
  * 現在2種類（シンプル版、ウィザード版）あるCSV取り込み画面を、使いやすい「ウィザード版」1つに統一します。
  * 古い画面や、重複しているコードを削除してスッキリさせます。
* **効果**
  * 「どっちを使えばいいの？」という迷いがなくなります。
  * 開発側もメンテナンスが楽になり、バグが減ります。

### ✅ フェーズ2：不具合の修繕

**「当たり前のことを当たり前にできる」**

* **何をする？**
  * 検索画面からデータを編集・削除できるように直します。
  * 自動分類ルールの設定画面を使いやすい方に統一します。
  * カテゴリ名の微妙な違い（"Amazon" と "Ａｍａｚｏｎ"など）を自動で同じものとして扱うようにします。
* **効果**
  * 「検索したけど直せない」といったストレスがなくなります。
  * データの分類がより賢くなります。

### ✅ フェーズ3：快適化・品質向上

**「データが増えてもサクサク動く」**

* **何をする？**
  * 残高計算の方法を見直し、データが数万件になっても遅くならないようにします。
  * グラフが無駄に何度も描画されるのを防ぎ、バッテリー消費を抑えます。
* **効果**
  * 長く使い続けてデータが溜まっても、アプリが重くなりません。
  * スマホの電池持ちが少し良くなります。

## 🚀 将来の追加機能（案）

土台が固まった後、さらに便利にするためのアイデアです。

1. **予算管理の強化**: 「今月あといくら使える？」を日割りで表示。
2. **固定費の自動化**: 月初に自動で家賃などを入力済みにしてくれる機能。
3. **資産推移グラフ**: 複数の口座の合計資産がどう増減しているかを可視化。
4. **レシート読取強化**: 買った品物ごとに細かくカテゴリ分けできるように改善。
5. **カレンダー改善**: 支出だけでなく、収入や収支（＋ー）もカレンダーで見れるように。

---

## 📝 まとめ

まずは **「フェーズ0（止血）」** で、アプリの安全性と信頼性を確保することから始めさせてください。
新機能の追加は、その土台ができてから行うことで、より高品質なアプリを最速で提供できるようになります。
