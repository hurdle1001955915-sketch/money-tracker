# ライブデモ操作フロー（更新版）

> 想定時間: 5〜7分
> iPhone画面共有（AirPlay / Zoom画面共有 / QuickTime Player等）

---

## 事前準備チェックリスト

- [ ] iPhoneに最新ビルドをインストール済み
- [ ] デモ用サンプルデータ（取引50件以上）が入っている
- [ ] ミラーリング接続テスト完了
- [ ] 通知OFF（集中モード or おやすみモード ON）
- [ ] 電池残量70%以上 / 充電ケーブル接続推奨
- [ ] インポート用CSVファイルを「ファイル」アプリに保存済み
- [ ] OpenAI APIキーが設定済み、AI分類が動作確認済み
- [ ] ネットワーク接続を確認（AI分類に必要）
- [ ] バックアップ用スクリーンショット / 画面収録を準備

---

## デモフロー

### Phase 1: ファーストインプレッション（1分）

**1. アプリ起動 → Face ID**
- アプリアイコンをタップ
- Face IDのロック画面が表示
- 「スライドで紹介した通り、セキュリティもしっかり実装しています」
- Face IDで認証 → アプリ表示
- **ポイント**: 「完全ローカルのアプリなので、Face IDで守られています」

**2. カレンダー画面をさっと見せる**
- 月間カレンダーが表示される
- 上部のサマリーバー（収入・支出・差額）を指差し
- 「これがメイン画面です。後で詳しく見せます」

---

### Phase 2: 取引入力（1分）

**3. 入力タブに移動**
- 「入力」タブをタップ
- 「日常の取引記録はこの画面です」

**4. 金額入力デモ**
- 電卓で「1280」と入力
- 「電卓スタイルなので直感的です」

**5. カテゴリ選択**
- カテゴリ一覧を表示
- 「よく使うカテゴリが上に来ています」
- 「食費」をタップ

**6. 保存**
- メモに「ランチ」と入力 → 保存
- 「数秒で記録完了です」

---

### Phase 3: データビジュアライゼーション（1分30秒）

**7. グラフタブ**
- 「グラフ」タブをタップ
- 支出の円グラフが表示
- 「カテゴリ別の割合が一目で分かります」

**8. グラフ切り替え（スワイプ）**
- 左にスワイプ → 収入円グラフ
- スワイプ → 貯蓄推移
- スワイプ → 年間支出棒グラフ
- 「7種類のグラフをスワイプで切り替えられます」
- **ポイント**: アニメーションの滑らかさを見せる

**9. カテゴリ詳細**
- 円グラフのスライスをタップ
- 「詳細と月ごとの推移が見れます」

**10. 資産ダッシュボード**
- 「資産」タブをタップ
- 「総資産額と口座別のポートフォリオです」
- 資産推移チャートを指差し

---

### Phase 4: CSV AI分類 -- クライマックス（2分）

> ここがデモの最大の見せ場。ゆっくり丁寧に。

**11. 設定 → CSVインポート**
- 「設定」タブ → 「CSVインポート」
- 「**ここからが今日一番見せたかった機能です**」

**12. CSVファイル選択**
- ファイル選択画面でCSVを選択
- 「PayPayの利用明細CSVを読み込みます」

**13. 自動フォーマット検出**
- 「見てください、**PayPay CSVとして自動検出されました**」
- 「ファイルの中身を分析して自動判定しています」

**14. プレビュー確認**
- ウィザードのプレビューを表示
- 「取り込まれるデータが確認できます」

**15. AI自動分類を実行** ★ハイライト
- 「AI自動分類」ボタンをタップ
- 「GPT-4o-miniにデータを送って、カテゴリを自動判定させます」
- プログレスバーが進む
- 「バッチ1... バッチ2... 完了！」

**16. 分類結果を見せる** ★最大の見せ場
- 結果一覧をスクロール
- 具体例を指差す:
  - 「"マクドナルド" → "外食"」
  - 「"セブンイレブン" → "コンビニ"」
- 「信頼度スコアも表示されています。95%、87%...」
- 「分類の**理由も日本語で**出ています」
- **ポイント**: ここで少し間を取る → 聴衆の反応を待つ

**17. インポート確定**
- 「これで数百件の取引が一気に取り込まれました」

---

### Phase 5: クロージング（30秒）

**18. カレンダーに戻る**
- インポートデータが反映されていることを確認
- 「以上がデモでした」

**19. 一言**
- 「このアプリは24,000行のコードで、その大半をAIと一緒に2〜3週間で作りました」
- 「データは全て、今このiPhoneの中だけに保存されています」

**20. Q&Aへ**
- 「ご質問があればお願いします！」

## バックアッププラン

| トラブル | 対処法 |
|---------|--------|
| ネットワーク障害（AI分類不可） | 事前に撮影したスクリーンショットで結果を見せる |
| AI分類が遅い（1分以上） | 「バッチ処理中です」と説明。事前収録に切り替え |
| アプリクラッシュ | 再起動して続行。最悪はスクショで説明 |
| 画面共有が切れる | QuickTime Player再接続。だめならiPhoneを直接見せる |
| CSVインポートエラー | 「形式不一致はスキップされます」と自動回復を説明 |

---

## デモ時の話し方のコツ

1. **操作と説明の間に少し間を空ける** -- 画面の切り替わりを聴衆が追えるように
2. **「見てください」を使う** -- 注目ポイントの前に言うと効果的
3. **数字を繰り返す** -- 「7種類」「6フォーマット」「80%の閾値」
4. **AI分類の瞬間を盛り上げる** -- 「ここからが今日一番のハイライトです」
5. **結果が出た瞬間に少し間を取る** -- 聴衆の「おおっ」を待つ
6. **最後にプライバシーを再強調** -- 「このデータは全てこのiPhoneの中だけです」
